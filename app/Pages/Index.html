Why CQRS instead of MVC?

MVC controllers grow into junk drawers. Every new feature finds a "natural home" alongside three unrelated actions, the file gets longer, the dependencies multiply, and a year later you cannot test one method without booting half the application.

CQRS keeps each operation small, named, and testable. Every read is a Query, every write is a Command, each one is its own DTO with its own handler — discovered by convention, validated by attribute, dispatched through middleware. Handlers stay tiny because the framework absorbs the ceremony. 

final class Health
{
    public function __construct(
        public readonly bool $verbose = false,
    ) {}
}

final class HealthHandler
{
    public function __invoke(Health $q): array
    {
        $result = ['status' => 'ok'];
        if ($q->verbose) {
            $result['php'] = PHP_VERSION;
        }
        return $result;
    }
}
final class SubscribeNewsletter
{
    public function __construct(
        #[Email] public readonly string $email,
    ) {}
}

final class SubscribeNewsletterHandler
{
    public function __construct(
        private readonly Subscribers $subscribers,
    ) {}

    public function __invoke(SubscribeNewsletter $c): void
    {
        $this->subscribers->add($c->email);
    }
}
